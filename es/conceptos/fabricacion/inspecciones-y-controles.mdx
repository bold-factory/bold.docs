Las **inspecciones y controles** capturan evidencias, validaciones y resultados de calidad. Pueden aparecer en producción, catálogo o almacén.

Úsalos cuando una comprobación deba quedar vinculada a una operación, artículo, recepción o movimiento real.

## Dónde aparecen

| Área | Qué valida |
| --- | --- |
| Producción | Controles de operación al inicio, durante el avance o al final. |
| Catálogo | Inspecciones asociadas al artículo para definir qué revisar. |
| Almacén | Inspecciones de recepción antes de registrar entrada física. |
| Mantenimiento | Acciones con resultado dentro de un parte de trabajo. |

No todos los controles son de calidad estricta. Algunos registran confirmaciones operativas, fotos, medidas o archivos que explican cómo se hizo el trabajo.

## Tipos de resultado

| Tipo | Cuándo usarlo |
| --- | --- |
| Confirmación | El operario debe confirmar que una acción se hizo. |
| Sí/no | Necesitas un resultado aceptado o rechazado. |
| Número | Quieres registrar una medida. |
| Texto | Necesitas una observación libre. |
| Imagen | Debe quedar evidencia visual. |
| Archivo | Debe adjuntarse un documento o prueba externa. |

Configura el tipo según la decisión que vas a tomar con el dato. Si nadie revisa el resultado, probablemente no necesitas un control obligatorio.

## Momento de ejecución

| Momento | Efecto |
| --- | --- |
| Al inicio | Puede bloquear el arranque si falta una validación previa. |
| Por unidad o progreso | Se registra durante el avance. |
| Al final | Puede bloquear el cierre hasta completar la comprobación. |

Los controles obligatorios protegen la trazabilidad. Si el operario no puede iniciar o finalizar, revisa qué inspección falta y si requiere archivo, foto, valor o confirmación.

## Inspecciones de recepción

Una recepción puede exigir inspecciones antes de registrar la entrada. Esto permite revisar calidad, cantidades, documentación o evidencias antes de aumentar stock.

Después de recibir, la recepción y sus inspecciones quedan protegidas para conservar el historial.

## Relación con costes y trazabilidad

Los controles no calculan coste por sí solos, pero explican desviaciones, rechazos, reprocesos o bloqueos.

Cuando una inspección genera rechazo, pausa o ajuste, conecta el resultado con [Controles, consumos y pausas](/es/conceptos/fabricacion/controles-consumos-y-pausas), [Stock y movimientos](/es/conceptos/almacen/stock-y-movimientos) o [Coste teórico vs coste real](/es/conceptos/costes/coste-teorico-vs-real).
