Datos y archivos cubren importaciones, documentos adjuntos, evidencias y mantenimiento de información usada por varios módulos.

## Importaciones

Las importaciones sirven para cargar datos maestros o históricos. Antes de importar, valida códigos, unidades, relaciones y duplicados.

| Riesgo | Revisión necesaria |
| --- | --- |
| Códigos duplicados | Confirma que cada producto, artículo, ubicación o activo tenga una referencia única. |
| Unidades incoherentes | Revisa unidades antes de crear stock, costes o recetas. |
| Relaciones incompletas | Comprueba productos, propiedades, clientes, proveedores o ubicaciones referenciadas. |
| Datos obsoletos | Separa carga inicial, corrección y actualización periódica. |

Una importación con códigos inconsistentes crea trabajo de limpieza en catálogo, planificación y stock.

## Archivos

Los archivos se usan para documentos, evidencias, imágenes, adjuntos de acciones, tareas con agentes o diseños. Mantén nombres claros para que los equipos encuentren el archivo correcto.

Algunos flujos requieren permisos de archivos además del permiso funcional. Por ejemplo, subir evidencias puede requerir permisos de carga de archivos.
