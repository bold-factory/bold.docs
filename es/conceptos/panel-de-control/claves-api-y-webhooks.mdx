Las **claves API** permiten que una integración actúe contra la API de Bold. Los **webhooks** avisan a sistemas externos cuando ocurre un evento.

## Claves API

| Concepto | Uso |
| --- | --- |
| Clave API | Credencial técnica para autenticar una integración. |
| Alcance | Permisos que determinan qué puede leer o modificar la clave. |
| Sistema externo | Aplicación que usa la clave para integrarse con Bold. |

Crea claves separadas por sistema externo. Así puedes revocar una integración sin afectar a las demás.

## Webhooks

| Concepto | Uso |
| --- | --- |
| Evento | Cambio relevante en Bold, como stock, pedido, orden o parte. |
| Suscripción | Configuración que decide qué eventos se envían y a qué destino. |
| Entrega | Petición HTTP enviada al sistema externo. |

Usa webhooks cuando otro sistema necesite reaccionar a cambios de catálogo, stock, pedidos, producción o mantenimiento.

<Tip>
  Para integraciones críticas, combina webhooks con lecturas periódicas paginadas. El webhook avisa rápido y la reconciliación detecta eventos perdidos o errores de entrega.
</Tip>

## Reglas

- Guarda la clave API al crearla. No la compartas entre sistemas externos.
- Concede solo los permisos necesarios para la integración.
- Valida que el endpoint de webhook pueda recibir reintentos y registrar errores.
- Usa la API para reconciliar datos cuando la entrega de eventos sea crítica.
