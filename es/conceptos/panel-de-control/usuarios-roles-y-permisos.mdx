Un **usuario** accede a Bold. Un **perfil** agrupa permisos. Un **permiso** habilita una acción concreta.

Estos conceptos controlan qué puede ver y cambiar cada persona desde el Panel de control.

## Entidades

| Concepto | Uso |
| --- | --- |
| Usuario | Identidad con credenciales de acceso. |
| Perfil | Conjunto reutilizable de permisos. |
| Permiso | Capacidad concreta para leer, editar o ejecutar una acción. |
| Empleado | Persona operativa que registra trabajo, aunque no siempre tenga usuario. |

Usuario y empleado pueden estar vinculados, pero no son lo mismo. El usuario responde a "quién entra y qué puede hacer". El empleado responde a "quién trabaja, registra tiempo o participa en costes".

## Diseño de perfiles

Crea perfiles por responsabilidad real, no por persona individual.

| Perfil típico | Alcance habitual |
| --- | --- |
| Administración | Configuración, usuarios y datos maestros. |
| Almacén | Stock, recepciones, movimientos e inventarios. |
| Producción | Órdenes, operaciones, consumos y progreso. |
| Mantenimiento | Activos, preventivos y partes. |
| Consulta | Lectura sin cambios operativos. |

Evita perfiles demasiado amplios para perfiles temporales o externos. Un permiso de escritura puede cambiar stock, costes o trazabilidad.

## Cómo se organizan los permisos

Los permisos suelen separar consulta, edición y ejecución. La consulta permite ver datos. La edición permite crear, modificar, completar o cancelar registros. La ejecución habilita acciones operativas específicas, como trabajar en planta, secuenciar órdenes o activar versiones.

Asigna permisos según la responsabilidad real de cada equipo. No todos los perfiles necesitan modificar datos, y algunas acciones críticas conviene reservarlas a responsables concretos.

## Permisos por módulo

Los permisos suelen separar lectura y escritura. La lectura permite consultar; la escritura permite crear, editar, completar o cancelar según el módulo.

Algunos flujos requieren permisos adicionales. Por ejemplo, subir evidencias puede requerir permisos de archivos además del permiso funcional.

| Módulo | Accesos habituales |
| --- | --- |
| Acceso común | Entrada a Bold, modo administrativo y modo de trabajo. |
| Catálogo | Productos, propiedades y artículos. |
| Almacén | Ubicaciones, recepciones, envíos, inventarios, movimientos y bultos. |
| Planificación | Clientes, proveedores, pedidos, recomendaciones y artículos planificables. |
| Producción | Recetas, órdenes, peticiones, estaciones y secuenciación. |
| Mantenimiento | Activos, causas, acciones, partes y preventivos. |
| Personal | Usuarios, empleados, turnos, presencia y tareas. |
| Archivos y etiquetas | Archivos adjuntos, evidencias, diseños y plantillas. |
| Informes y agentes | Informes, Boldy, agentes y tareas delegadas. |

Administrar usuarios, perfiles y permisos requiere un perfil de administración de la aplicación. No lo gestiones como un permiso operativo de personal.

<Tip>
  Si una acción no aparece o falla, revisa dos cosas: el estado del documento y los permisos del usuario.
</Tip>

## Claves de integración y agentes

Las claves de integración y los agentes también necesitan alcance controlado. Usa credenciales separadas por integración y concede a cada agente solo los permisos que necesita para sus tareas.
