Las **notificaciones** ayudan a detectar eventos relevantes sin revisar cada módulo manualmente. Están orientadas a personas y se basan en eventos de Bold.

Consulta [Auditoría y eventos](/es/conceptos/panel-de-control/auditoria-y-eventos) para entender de dónde salen los eventos que alimentan notificaciones, avisos externos y automatizaciones.

## Canales

| Canal | Uso |
| --- | --- |
| Aplicación | Avisos dentro de Bold con contador, lista y estado de lectura. |
| Correo electrónico | Correos para eventos que deben verse aunque el usuario no esté conectado. |
| Aviso externo | Entregas para sistemas externos. Se administra como integración técnica. |

Cada usuario gestiona sus propias suscripciones cuando el canal lo permite.

## Qué conviene notificar

Configura notificaciones para cambios que requieren acción, como recepciones completadas, envíos, órdenes, partes de mantenimiento o eventos de administración.

Evita notificar cada cambio informativo. Demasiado ruido hace que los avisos importantes se ignoren.

## Sesiones de agente

Cuando trabajas con [agentes de Smart Factory](/es/conceptos/smart-factory/agentes), puedes recibir avisos en la aplicación y por correo cuando:

- El agente publica una respuesta en una sesión.
- El agente pide una aclaración con preguntas u opciones para poder continuar.

El correo incluye un enlace para abrir la sesión y responder. Estos avisos también están disponibles como eventos para integraciones; consulta [Eventos disponibles](/es/desarrolladores/webhooks/eventos-disponibles) si necesitas reaccionar desde un sistema externo.

## Relación con permisos

Una notificación no debe sustituir permisos de lectura. Si el usuario no tiene acceso al módulo o registro, puede recibir menos contexto o no poder abrir el detalle.
