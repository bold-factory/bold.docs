Los **clientes** y **proveedores** identifican con quién se generan demanda, suministro, referencias externas y condiciones operativas.

## Clientes

Un cliente da contexto a pedidos de venta, entregas, referencias externas y análisis comercial.

| Dato | Uso |
| --- | --- |
| Código | Identificador estable. |
| Nombre | Búsqueda y comunicación. |
| Referencia externa | Integración con ERP, CRM o ecommerce. |
| Dirección o datos fiscales | Entrega, facturación o documentación. |

## Proveedores

Un proveedor da contexto a compras, plazos, precios y referencias de artículos.

| Dato | Uso |
| --- | --- |
| Proveedor principal | Ayuda a recomendar compras. |
| Código de proveedor | Relaciona el artículo con el catálogo externo. |
| Lead time | Anticipa cuándo comprar. |
| Cantidad mínima | Ajusta recomendaciones. |

## Relación con artículos

Para que una recomendación de compra sea útil, no basta con crear el proveedor. Debe existir una relación entre proveedor y artículo con los datos de suministro relevantes.
