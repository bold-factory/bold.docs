Las **habilidades** y las **estaciones habituales** describen qué trabajo suele poder hacer un empleado y dónde trabaja normalmente.

No sustituyen los permisos de usuario. Los permisos controlan el acceso a Bold. Las habilidades y estaciones habituales ayudan a organizar la ejecución operativa.

## Diferencias clave

| Concepto | Qué representa |
| --- | --- |
| Habilidad | Un proceso o tipo de operación que el empleado puede ejecutar. |
| Estación habitual | Una estación de trabajo en la que el empleado trabaja normalmente. |
| Asignación | Una operación o grupo concreto preparado para una persona, estación o momento. |
| Permiso de usuario | Capacidad de entrar en módulos y acciones de la aplicación. |

Un empleado puede tener habilidades sin tener usuario. También puede tener usuario sin estar configurado para trabajo operativo.

## Habilidades

Una habilidad se basa en un [proceso](/es/conceptos/fabricacion/procesos). Por ejemplo, un empleado puede estar habilitado para **Corte**, **Montaje** o **Inspección final**.

Usa habilidades cuando quieras:

- mostrar trabajo compatible con la capacidad del empleado;
- evitar que una persona tenga que buscar entre procesos que no realiza;
- analizar mejor qué perfiles cubren cada tipo de trabajo;
- preparar asignaciones más coherentes.

La habilidad indica capacidad operativa. No garantiza que el empleado esté disponible en ese momento.

## Estaciones habituales

Una estación habitual indica dónde trabaja normalmente una persona. Sirve para priorizar operaciones cercanas a su puesto o célula de trabajo.

Usa estaciones habituales cuando:

- un empleado trabaja casi siempre en una misma estación;
- una célula de trabajo tiene operarios recurrentes;
- quieres reducir listas largas en el modo de trabajo;
- necesitas orientar tareas por zona antes que por empleado asignado.

La estación habitual no bloquea trabajar en otra estación. La asignación real de la operación sigue marcando dónde se ejecuta.

## Impacto en modo de trabajo

En [Modo de trabajo](/es/conceptos/modo-de-trabajo), el empleado activo puede ver tareas según:

| Criterio | Cómo ayuda |
| --- | --- |
| Habilidades | Muestra procesos que el empleado sabe ejecutar. |
| Estaciones habituales | Prioriza trabajo de sus estaciones normales. |
| Operaciones asignadas | Enseña trabajo preparado para ese empleado. |
| Grupos de operaciones | Permite continuar bloques de trabajo compartidos. |
| Operaciones activas o recientes | Facilita reanudar trabajo iniciado. |

Configura habilidades y estaciones habituales con el mismo nivel de detalle que uses en planta. Si defines demasiadas, la selección pierde valor. Si defines muy pocas, el empleado verá trabajo poco relevante.
