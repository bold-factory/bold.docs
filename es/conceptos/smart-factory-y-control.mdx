Smart Factory agrupa análisis, asistentes y automatizaciones. El panel de control concentra configuración de cuenta, acceso, datos, eventos y diseños.

## Smart Factory

| Concepto | Uso |
| --- | --- |
| Informes | Informes Power BI embebidos para analizar operación, producción, almacén o costes. |
| Boldy | Conversaciones para consultar información y recibir ayuda contextual. |
| Tareas con agentes | Trabajos ejecutados por agentes con historial, estado y consumo de créditos. |
| Nivel de inteligencia | Configuración que ajusta profundidad, coste y tiempo de respuesta de un agente. |

Las conversaciones con Boldy sirven para interacción directa. Las tareas con agentes sirven para trabajos más largos, trazables o automatizados.

Los informes Power BI sirven para análisis periódico y dirección. No sustituyen a las vistas operativas en tiempo real, como cuadros de mando de fabricación o pantallas de ejecución.

## Agentes

Un agente combina instrucciones, alcance, automatizaciones y nivel de inteligencia. Úsalo para tareas repetibles o análisis que requieren varios pasos.

| Nivel | Cuándo usarlo |
| --- | --- |
| Bajo | Consultas directas, resúmenes simples o tareas muy acotadas. |
| Medio | Tareas operativas habituales con equilibrio entre coste y razonamiento. |
| Alto | Análisis con varios pasos, instrucciones ambiguas o decisiones con muchos matices. |

Empieza por el nivel más bajo que cumpla la tarea. Sube el nivel si el agente omite pasos, no entiende el contexto o necesita razonar con varias áreas.

## Panel de control

| Área | Qué gestiona |
| --- | --- |
| Cuenta y plan | Datos de suscripción, límites y uso visible para administradores. |
| Empresa | Datos generales y configuración de negocio. |
| Acceso | Usuarios, roles, permisos y claves API. |
| Eventos | Webhooks, notificaciones y automatizaciones. |
| Datos | Importaciones, archivos y mantenimiento de información. |
| Diseños | Plantillas, etiquetas y formatos de salida. |
| Agentes | Configuración de agentes, automatizaciones y nivel de inteligencia. |

## Claves API y webhooks

Una clave API permite que una integración actúe contra la API de Bold. Crea claves separadas por sistema externo para poder revocarlas de forma independiente.

Un webhook avisa a un sistema externo cuando ocurre un evento. Úsalo cuando otro sistema necesite reaccionar a cambios de catálogo, stock, pedidos, producción o mantenimiento.

<Tip>
  Para integraciones críticas, combina webhooks con lecturas periódicas paginadas. El webhook avisa rápido y la reconciliación detecta eventos perdidos o errores de entrega.
</Tip>

## Importación y archivos

Las importaciones sirven para cargar datos maestros o históricos. Antes de importar, valida códigos, unidades, relaciones y duplicados. Una importación con códigos inconsistentes crea trabajo de limpieza en catálogo, planificación y stock.

Los archivos y diseños se usan para documentos, etiquetas o evidencias. Mantén nombres y plantillas claros para que los equipos encuentren el formato correcto.

## Diseños y plantillas de etiquetas

Las plantillas de etiquetas permiten imprimir identificadores para artículos, lotes, estaciones u otros objetos. Pueden incluir texto, códigos QR, códigos de barras y valores dinámicos de Bold.

Una plantilla debe ser legible en el tamaño real de impresión y contener solo la información necesaria para identificar el objeto en planta o almacén. Antes de usarla de forma masiva, descarga una muestra y pruébala con el lector que usará el equipo.

## Notificaciones

Las notificaciones ayudan a detectar eventos relevantes sin revisar cada módulo manualmente. Se basan en el mismo catálogo de eventos que los webhooks, pero están orientadas a personas.

Cada usuario gestiona sus propias suscripciones por canal:

| Canal | Uso |
| --- | --- |
| App | Avisos dentro de Bold con contador, lista y estado de lectura. |
| Email | Correos para eventos que deben verse aunque el usuario no esté conectado. |
| Webhook | Entregas HTTP para sistemas externos. Se administra de forma separada. |

Configura notificaciones para cambios que requieren acción, como recepciones completadas, envíos, órdenes, partes de mantenimiento o eventos de administración.
