El catálogo es la base de Bold. Define qué se compra, fabrica, almacena, consume, vende y valora.

## Entidades principales

| Concepto | Uso |
| --- | --- |
| Producto | Familia comercial o técnica que agrupa artículos relacionados. Define las propiedades que deben informar sus artículos. |
| Artículo | Elemento físico único. Bold controla sobre él stock, compras, ventas, fabricación, lotes y costes. |
| Propiedad | Atributo reutilizable que diferencia variantes, como color, medida, material o acabado. |
| Valor de propiedad | Valor concreto que convierte un artículo en una variante válida de un producto. |
| Lote | Conjunto trazable de unidades de un artículo. Conserva origen, fabricación, calidad o coste. |
| Co-producto | Salida adicional generada durante una fabricación. |

## Producto y artículo

Un **producto** agrupa variantes. Un **artículo** identifica la variante operativa.

Por ejemplo, **Mesa Nórdica** puede ser el producto. **Mesa Nórdica - 160 cm - Roble** y **Mesa Nórdica - 200 cm - Nogal** son artículos distintos porque tienen stock, pedidos y fabricación separados.

El backend valida código y nombre al crear productos y artículos. Cuando un producto tiene propiedades asignadas, cada artículo de ese producto debe informar un valor para todas ellas. Si falta un valor, el artículo queda **inválido** hasta completar la información.

## Propiedades

Las propiedades se crean una vez y se reutilizan en productos, recetas y condiciones. Pueden representar texto, número, fecha, sí/no u opciones cerradas.

| Clase | Ejemplo | Regla |
| --- | --- | --- |
| Texto | Acabado libre | Acepta texto. |
| Número | Longitud | Debe ser numérico. |
| Fecha | Fecha de homologación | Debe tener formato de fecha. |
| Sí/no | Ignífugo | Debe ser verdadero o falso. |
| Selección | Color | Debe usar una opción existente. |

<Warning>
  No uses texto libre para datos que condicionan recetas o planificación. Usa opciones cerradas para evitar variantes duplicadas.
</Warning>

## Validación y consolidación

Un artículo sin producto se considera válido. Un artículo con producto se valida contra las propiedades del producto.

Bold puede proponer consolidar artículos cuando tienen el mismo producto y los mismos valores de propiedades. Consolidar deja un artículo como referencia válida y retira el duplicado. Úsalo solo si ambos registros representan exactamente el mismo elemento físico.

El backend solo permite fusionar artículos del mismo producto con el mismo conjunto de valores. La fusión emite eventos de consolidación y elimina de forma lógica el artículo absorbido.

## Lotes

Usa lotes cuando necesites trazabilidad por compra, fabricación, origen, caducidad, control de calidad o coste. Un lote se combina con artículo y ubicación para explicar dónde está cada cantidad.

Gestionar múltiples lotes implica seleccionar el lote correcto al recibir, consumir, mover o enviar material. Si un artículo no requiere trazabilidad por lote, puedes operar con stock sin lote.

## Relación con otros módulos

| Módulo | Dependencia del catálogo |
| --- | --- |
| Planificación | Las líneas de demanda y suministro necesitan artículos válidos, plazos, punto de pedido y origen de aprovisionamiento. |
| Almacén | El stock se registra por artículo, ubicación y lote opcional. |
| Fabricación | Las recetas se definen a nivel de producto y producen o consumen artículos. |
| Costes | El coste se calcula sobre artículos, lotes, consumos, producción y valor de inventario. |
| API | Las integraciones pueden usar `sku` para referirse al artículo operativo. |
