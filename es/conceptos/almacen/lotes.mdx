Un **lote** identifica un conjunto trazable de unidades de un mismo artículo. Aunque dos unidades pertenezcan al mismo artículo, pueden diferir por proveedor, fecha, fabricación, coste, calidad u origen.

## Cuándo usar lotes

Usa lotes si necesitas:

- trazabilidad de compra o fabricación;
- control de calidad por origen;
- separación por caducidad, fecha o proveedor;
- valoración por coste unitario real;
- investigación de incidencias;
- relación entre componentes consumidos y producto fabricado.

Si tu operación no necesita separar unidades por origen, puedes mantener stock sin lote para ese artículo.

## Qué contiene un lote

| Dato | Uso |
| --- | --- |
| Código | Identifica el lote en operaciones y consultas. |
| Artículo | Define qué unidad física contiene. |
| Origen | Compra, fabricación, ajuste, fusión u otro movimiento. |
| Stock actual | Cantidad restante por ubicación. |
| Coste y valor | Permiten valorar inventario y analizar diferencias. |
| Relaciones | Documentos, recepciones, consumos, órdenes o controles asociados. |

Un lote está activo mientras conserva stock o historial relevante.

## Tipos de lote

Bold distingue tipos de lote según cómo se creó y qué trazabilidad conserva.

| Tipo | Cuándo aparece |
| --- | --- |
| Lote de origen | Nace desde una compra, fabricación u origen conocido. |
| Lote fusionado | Agrupa stock de dos lotes cuando decides dejar de separarlos. |
| Lote provisional | Permite conservar una referencia cuando falta información completa de origen. |

El tipo de lote ayuda a interpretar su historial. Un lote de origen conserva una procedencia directa. Un lote fusionado conserva relación con los lotes que lo componen.

## Origen del lote

El origen indica de dónde viene el material trazable.

| Origen | Significado |
| --- | --- |
| Compra | El lote nace de una entrada de proveedor. |
| Fabricación | El lote nace de una orden o progreso productivo. |
| Desconocido | El lote existe, pero no hay origen operativo fiable. |

Evita usar origen desconocido como solución normal. Úsalo solo cuando el historial real no esté disponible.

## Relación con almacén

Bold registra stock por artículo, ubicación y lote opcional. Si un artículo trabaja con lotes, cada recepción, consumo, movimiento o envío debe conservar el lote correcto.

La trazabilidad pierde valor si el equipo usa un lote genérico para registrar salidas. El lote debe coincidir con el material físico que se mueve o consume.

## Relación con costes

El lote puede conservar coste unitario y origen económico. Esto permite valorar inventario por compra, fabricación o regularización.

Dos lotes del mismo artículo pueden tener costes distintos. Esa diferencia es normal si proceden de compras, fabricaciones o ajustes distintos.

## Fusionar lotes

Fusiona lotes solo cuando ya no necesitas mantener separado su origen. La fusión simplifica operación futura, pero reduce la granularidad de análisis.

No fusiones lotes si necesitas investigar calidad, proveedor, fecha de fabricación, caducidad o coste por origen.

Cuando fusionas lotes, el lote resultante conserva relación con los lotes de origen y sus cantidades. A partir de ese momento, la operación diaria trabaja con el lote fusionado.
