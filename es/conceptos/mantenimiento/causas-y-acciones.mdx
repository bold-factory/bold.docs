Las **causas** explican por qué se realiza una intervención. Las **acciones** explican qué trabajo o comprobación debe realizarse.

## Causas

Una causa clasifica el motivo principal o auxiliar de un parte.

| Uso | Ejemplo |
| --- | --- |
| Avería | Rotura, fuga, atasco. |
| Preventivo | Revisión periódica, lubricación. |
| Mejora | Ajuste, modificación, cambio de formato. |
| Seguridad | Inspección, corrección obligatoria. |

Un parte puede tener una causa principal y causas auxiliares. Las auxiliares no deben duplicar la principal.

## Acciones

Una acción es una tarea reutilizable o comprobación que puede completarse en un parte.

| Tipo | Resultado esperado |
| --- | --- |
| Simple | No requiere dato adicional. |
| Sí/no | Valor booleano. |
| Número | Medida decimal. |
| Texto | Observación. |
| Imagen o archivo | Evidencia adjunta. |

Define acciones cuando quieres estandarizar trabajo, medir cumplimiento o comparar intervenciones.

## Relación entre causa y acción

Una causa puede sugerir acciones habituales. Esto ayuda a que los partes incluyan comprobaciones coherentes para un tipo de incidencia.

Ejemplo: la causa **Fuga hidráulica** puede sugerir acciones como **Revisar latiguillo**, **Comprobar presión** y **Limpiar zona**.
