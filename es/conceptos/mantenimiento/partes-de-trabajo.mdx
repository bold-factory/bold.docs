Un **parte de trabajo** registra una intervención de mantenimiento sobre un activo.

Explica qué ocurrió, por qué ocurrió, quién intervino, cuánto tiempo se dedicó, qué acciones se completaron y qué materiales se consumieron.

## Qué contiene

| Dato | Uso |
| --- | --- |
| Activo | Equipo o instalación intervenida. |
| Tipo | Correctivo, preventivo u otro criterio operativo. |
| Causa | Motivo principal y posibles causas auxiliares. |
| Acciones | Tareas o comprobaciones realizadas. |
| Tiempos | Mano de obra por empleado. |
| Materiales | Repuestos o consumibles usados. |
| Estado | Situación del parte. |

## Estados habituales

| Estado | Significado |
| --- | --- |
| Pendiente | El parte existe, pero no tiene tiempo activo. |
| En progreso | Hay un registro de tiempo activo. |
| Pausado | Hubo trabajo, pero no hay tiempo activo. |
| Completado | La intervención queda cerrada y protegida. |
| Cancelado | El parte queda cerrado sin ejecución válida. |

## Materiales y trazabilidad

Los materiales consumidos vinculan mantenimiento con almacén y costes. Si el artículo trabaja con lotes, registra el lote real usado.

El coste del parte depende de materiales, lotes, cantidades, tiempo y coste horario de empleados.

## Partes preventivos

Una programación preventiva puede generar partes automáticamente. El parte generado sigue siendo la ejecución real; la programación solo define cuándo debe aparecer.
