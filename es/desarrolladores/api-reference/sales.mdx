---
title: "Sales"
description: "Referencia funcional de la API para ventas, clientes, pedidos y envíos."
---

Las ventas combinan recursos de `planning` y `warehouse`. En planificación creas la demanda de cliente. En almacén registras la salida física.

## Flujo de integración

<Steps>
  <Step title="Sincroniza clientes">
    Crea o actualiza clientes antes de enviar pedidos de venta.
  </Step>
  <Step title="Crea el pedido de venta">
    Añade cabecera, líneas de artículo, cantidades y fechas comprometidas.
  </Step>
  <Step title="Confirma líneas">
    Las líneas confirmadas generan demanda y afectan stock disponible.
  </Step>
  <Step title="Registra envío">
    Cuando el material sale, crea o completa un envío para reducir stock.
  </Step>
</Steps>

## Endpoint families

```http
GET    /v1/planning/customers
POST   /v1/planning/customers

GET    /v1/planning/salesOrders
POST   /v1/planning/salesOrders
GET    /v1/planning/salesOrders/{orderReference}
POST   /v1/planning/salesOrders/{orderReference}/lines/{lineReference}/confirm

GET    /v1/warehouse/shipments
POST   /v1/warehouse/shipments
POST   /v1/planning/salesOrders/{orderReference}/shipment
POST   /v1/warehouse/shipments/{shipmentReference}/ship
```

## Estados de venta

| Estado | Significado |
| --- | --- |
| `Draft` | La línea no genera demanda todavía. |
| `Pending` | La línea está confirmada y pendiente de entregar. |
| `PartiallyDelivered` | Se entregó parte de la cantidad. |
| `Delivered` | La línea quedó completamente entregada. |
| `Cancelled` | La línea ya no se entregará. |

## Permisos

| Permiso | Uso |
| --- | --- |
| `Planning.Customers.Read` / `Planning.Customers.Write` | Consultar o mantener clientes. |
| `Planning.SalesOrders.Read` / `Planning.SalesOrders.Write` | Consultar o gestionar pedidos de venta. |
| `Warehouse.Shipments.Read` / `Warehouse.Shipments.Write` | Consultar o registrar salidas de material. |
| `Warehouse.Skus.Read` | Consultar disponibilidad por artículo. |

<Warning>
  Confirmar líneas crea demanda. Enviar material reduce stock. Mantén esas acciones alineadas con el flujo físico para evitar disponibilidad incorrecta.
</Warning>
