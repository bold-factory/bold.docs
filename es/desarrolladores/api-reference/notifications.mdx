## Familias de endpoints

```http
GET    /v1/notifications/events
GET    /v1/notifications/events/versions

GET    /v1/notifications/app/subscriptions
PUT    /v1/notifications/app/subscriptions/{eventName}
DELETE /v1/notifications/app/subscriptions/{eventName}

GET    /v1/notifications/email/subscriptions
PUT    /v1/notifications/email/subscriptions/{eventName}
DELETE /v1/notifications/email/subscriptions/{eventName}

GET    /v1/notifications/app
POST   /v1/notifications/app/read
POST   /v1/notifications/app/{id}/read
POST   /v1/notifications/app/{id}/unread
```

Los endpoints de eventos requieren acceso a la aplicaciĂłn. Las suscripciones de app y email usan el email del usuario autenticado.

<Warning>
  No construyas nombres de eventos a mano. Consulta `/v1/notifications/events` o `/v1/notifications/events/versions` en el entorno donde vas a operar.
</Warning>

## Eventos

`GET /v1/notifications/events` devuelve nombres de evento.

```json
{
  "results": [
    {
      "eventName": "Warehouse.Receipt.Received"
    }
  ]
}
```

`GET /v1/notifications/events/versions` aĂ±ade la versiĂłn que deben usar integraciones sensibles al contrato.

```json

  "results": [
    {
      "eventName": "Warehouse.Receipt.Received",

    }
  ]
}
```

## Suscripciones personales

Cada canal tiene su propio conjunto de suscripciones. Activar **App** no activa **Email**, y activar **Email** no crea avisos dentro de Bold.

| Campo | DescripciĂłn |
| --- | --- |
| `id` | Identificador de la suscripciĂłn. |
| `email` | Email del usuario autenticado. |
| `eventName` | Nombre exacto del evento suscrito. |

Ejemplo para activar notificaciones de app:

```http
PUT /v1/notifications/app/subscriptions/Warehouse.Receipt.Received
```

Ejemplo para activar email:

```http
PUT /v1/notifications/email/subscriptions/Warehouse.Receipt.Received
```

Para desactivar una suscripciĂłn, usa el mismo `eventName` con `DELETE`.

## Bandeja de app

`GET /v1/notifications/app` devuelve notificaciones de app para el email del usuario autenticado.

| Campo | DescripciĂłn |
| --- | --- |
| `results` | Lista paginada de notificaciones. |
| `pageNumber` | PĂˇgina devuelta. |
| `totalPages` | Total de pĂˇginas disponibles. |
| `totalCount` | Total de notificaciones. |
| `totalUnreadCount` | Total de notificaciones sin leer. |

Cada notificaciĂłn incluye:

| Campo | DescripciĂłn |
| --- | --- |
| `id` | Identificador de la notificaciĂłn de app. |
| `eventName` | Evento que generĂł la notificaciĂłn. |
| `entity` | Tipo de entidad afectada. |
| `entityId` | Identificador de la entidad afectada. |
| `createdAt` | Fecha del evento de negocio. |
| `readAt` | Fecha de lectura. Es `null` si sigue sin leer. |

```json
{
  "results": [
    {
      "id": "ian_123",
      "eventName": "Warehouse.Receipt.Received",
      "entity": "Warehouse.Receipt",
      "entityId": "REC-00042",
      "createdAt": "2026-06-25T10:30:00Z",
      "readAt": null
    }
  ],
  "pageNumber": 1,
  "totalPages": 1,
  "totalCount": 1,
  "totalUnreadCount": 1
}
```

## Estados de lectura

Usa los endpoints de lectura para reflejar acciones de usuario en una bandeja personalizada.

```http
POST /v1/notifications/app/{id}/read
POST /v1/notifications/app/{id}/unread
POST /v1/notifications/app/read
```

`POST /v1/notifications/app/read` marca como leĂ­das las notificaciones pendientes.
