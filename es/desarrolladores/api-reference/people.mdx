El módulo `people` separa acceso a la aplicación, personas operativas y programación. Expone recursos para usuarios, roles, empleados, jornadas laborales, calendarios locales, turnos, asignaciones de calendario y períodos de trabajo calculados.

## Permisos

| Permiso o rol | Uso |
| --- | --- |
| `People.Users.Read` | Consultar usuarios. |
| `App.Admin` | Crear, editar o eliminar usuarios, roles y permisos asignados. |
| `People.Employees.Read` / `People.Employees.Write` | Consultar o gestionar empleados. |
| `Files.Files.Upload` + `People.Employees.Write` | Subir imagen de empleado. |
| `People.ShiftScheduling.Read` / `People.ShiftScheduling.Write` | Consultar o modificar jornadas, calendarios, turnos, programaciones y períodos. |
| `People.Tasks.Read` | Consultar tareas de personas. |

## Usuarios, roles y empleados

Un usuario controla acceso. Un empleado representa trabajo operativo. Pueden estar vinculados, pero son recursos distintos.

| Recurso | Campos principales |
| --- | --- |
| Usuario | `id`, `firstName`, `lastName`, `email`, roles y permisos cuando se consulta el detalle. |
| Rol | `id`, `name`, `key`, `description`, `systemRole` y permisos asignados. |
| Empleado | `id`, `externalReference`, `code`, `firstName`, `lastName`, `fullName`, `emails`, `mainUserEmail`, `pictureFileId`, auditoría y baja lógica. |

<Warning>
  No uses endpoints de empleados para conceder acceso. El acceso se gestiona con usuarios, roles y permisos.
</Warning>

## Jornadas, calendarios y turnos

Una jornada laboral contiene períodos con hora de inicio, hora de fin, cruce de medianoche y horas calculadas.

```json
{
  "name": "Noche",
  "periods": [
    {
      "startTime": "22:00:00",
      "endTime": "06:00:00",
      "endNextDay": true
    }
  ]
}
```

Un turno (`shiftSchedule`) contiene segmentos. Cada segmento referencia una jornada y dura un número de días.

```json
{
  "name": "Mañana y descanso",
  "description": "Cinco días de mañana y dos de descanso",
  "segments": [
    {
      "workDayTemplateId": "wdt_001",
      "durationDays": 5
    },
    {
      "workDayTemplateId": "wdt_rest",
      "durationDays": 2
    }
  ]
}
```

## Programación efectiva

La programación efectiva de un empleado combina turno, calendario local y asignaciones manuales.

| Prioridad | Fuente |
| --- | --- |
| 1 | Asignación manual de una fecha en `/schedule/dates`. |
| 2 | Fecha del calendario local asignado al empleado. |
| 3 | Patrón repetitivo del turno asignado al empleado. |

`GET /v1/people/workPeriods` devuelve períodos calculados con `employeeId`, `from`, `to` y `durationHours`. Usa `POST /v1/people/workPeriods/{employeeReference}/recalculate` cuando cambies reglas y necesites recalcular los períodos futuros de un empleado.
