El módulo `maintenance` gestiona activos mantenibles, acciones reutilizables, causas, partes de trabajo, tiempos, materiales, programaciones preventivas y auditoría.

## Estados y tipos

| Entidad | Valores principales |
| --- | --- |
| Parte de trabajo | `Pending`, `InProgress`, `Paused`, `Completed`, `Cancelled` |
| Tipo de parte | `Corrective`, `Preventive` |
| Tipo de acción | `Simple`, `Boolean`, `Number`, `Text`, `Image`, `File` |
| Política preventiva | `AllowEachOccurrence`, `SkipWhileOpen` |
| Tipo de disparador preventivo | `Periodic` |

## Permisos

| Permiso | Uso |
| --- | --- |
| `Maintenance.Assets.Read` / `Maintenance.Assets.Write` | Consultar o gestionar activos. |
| `Maintenance.ActionDefinitions.Read` / `Maintenance.ActionDefinitions.Write` | Consultar o gestionar acciones reutilizables. |
| `Maintenance.Causes.Read` / `Maintenance.Causes.Write` | Consultar o gestionar causas y acciones planificadas. |
| `Maintenance.WorkOrders.Read` / `Maintenance.WorkOrders.Write` | Consultar o gestionar partes, acciones, tiempos y materiales. |
| `Maintenance.PreventiveSchedules.Read` / `Maintenance.PreventiveSchedules.Write` | Consultar o gestionar programaciones preventivas. |
| `Files.Files.Upload` | Subir resultados de acciones de tipo imagen o archivo. |
| `App.Admin` | Consultar auditoría de mantenimiento. |

<Warning>
  Un parte completado o cancelado bloquea cambios de cabecera, acciones, tiempos y materiales. Para acciones de tipo imagen o archivo, usa los endpoints de subida específicos; completar con una URI manual solo es válido si cumple las reglas del tipo de resultado.
</Warning>

## Reglas de integración

| Regla | Impacto |
| --- | --- |
| Causas auxiliares | No pueden repetir la causa principal ni duplicarse entre sí. |
| Acciones completadas | No se pueden completar dos veces. |
| Tiempos | Añadir un tiempo activo inicia o reanuda el parte. Al cerrar todos los tiempos, el parte queda en pausa. |
| Materiales | Las cantidades deben ser positivas. Los detalles por lote no pueden duplicar lote ni superar la cantidad del material. |
| Preventivos | Programación y fecha de ocurrencia deben viajar juntas en partes generados. |
| Duplicados preventivos | Bold no crea dos partes no cancelados para la misma programación y ocurrencia. |
