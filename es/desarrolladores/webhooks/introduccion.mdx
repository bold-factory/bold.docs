---
title: "Introducción"
description: "Recibe notificaciones de eventos de Bold en sistemas externos."
---

Los webhooks envían una notificación HTTP cuando ocurre un evento en Bold. Úsalos para que un ERP, una herramienta de BI, una automatización interna o un middleware reaccionen sin consultar la API continuamente.

<Info>
  Los webhooks complementan la API. Usa la API para consultar o corregir el estado final, y usa webhooks para enterarte de que algo cambió.
</Info>

## Qué configura una suscripción

| Campo | Descripción |
| --- | --- |
| Evento | Nombre del evento que quieres recibir. |
| Versión | Versión del contrato del evento. |
| Endpoint | URL pública donde tu sistema recibe la notificación. |

```json
{
  "endpoint": "https://integraciones.example.com/bold/webhooks",
  "eventName": "SalesOrderLineConfirmed",
  "eventVersion": 1
}
```

## Flujo

<Steps>
  <Step title="Crea el endpoint receptor">
    Publica una URL HTTPS que acepte peticiones JSON y responda rápido.
  </Step>
  <Step title="Crea la suscripción">
    Registra `endpoint`, `eventName` y `eventVersion` en Bold.
  </Step>
  <Step title="Procesa el payload">
    Guarda el identificador del evento para evitar duplicados y usa la API si necesitas más datos.
  </Step>
  <Step title="Devuelve una respuesta correcta">
    Responde con un código `2xx` cuando el evento se acepte para procesamiento.
  </Step>
</Steps>

## Endpoints de suscripción

```http
GET /v1/notifications/webhook/subscriptions
POST /v1/notifications/webhook/subscriptions
DELETE /v1/notifications/webhook/subscriptions/{id}
```

Estas operaciones requieren permisos de administración de la aplicación.

<Warning>
  No ejecutes trabajo pesado dentro de la respuesta HTTP del webhook. Encola el evento, responde `2xx` y procesa en segundo plano.
</Warning>
