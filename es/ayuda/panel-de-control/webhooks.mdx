Los **webhooks** envían eventos de Bold a una URL externa. Úsalos cuando un ERP, BI, middleware o automatización interna debe reaccionar a cambios de pedidos, stock, fabricación, mantenimiento u otros datos.

## Qué gestiona la pantalla

| Pestaña | Uso |
| --- | --- |
| Suscripciones | Define qué evento y versión se envían a cada URL. |
| Notificaciones | Muestra entregas generadas, estado, intentos y errores. |

Gestionar webhooks requiere rol `App.Admin`.

<Info>
  Los webhooks avisan de que algo cambió. Si tu integración necesita el estado actual, usa la API después de recibir el evento.
</Info>

## Crear una suscripción

<Steps>
  <Step title="Prepara el receptor">
    Publica una URL que acepte `POST` JSON. Bold acepta `http` y `https`, pero usa `https` en producción.
  </Step>
  <Step title="Abre Webhooks">
    Entra en **Panel de control** y abre **Webhooks**.
  </Step>
  <Step title="Añade una suscripción">
    En **Suscripciones**, selecciona **Añadir suscripción**.
  </Step>
  <Step title="Elige el evento">
    Selecciona el evento disponible. Bold carga nombre y versión desde `/v1/notifications/events/versions`.
  </Step>
  <Step title="Indica la URL">
    Escribe el endpoint receptor y guarda la suscripción.
  </Step>
  <Step title="Prueba el flujo">
    Genera un evento de prueba funcional y revisa la pestaña **Notificaciones**.
  </Step>
</Steps>

## Revisar entregas

Cada notificación muestra evento, entidad, fecha, versión, estado, URL y número de intentos.

| Estado | Qué hacer |
| --- | --- |
| `Pending` | Espera al próximo intento. Revisa **Próximo intento** si aparece. |
| `Success` | El endpoint respondió con un código `2xx`. |
| `Failure` | Se agotaron los intentos automáticos. Abre el detalle y revisa los intentos. |

En el detalle puedes consultar el payload y la tabla de intentos. Cada intento conserva código HTTP, cuerpo de respuesta o mensaje de error.

## Reintentar una notificación fallida

<Steps>
  <Step title="Abre la notificación">
    En **Notificaciones**, abre una entrega con estado `Failure`.
  </Step>
  <Step title="Revisa el error">
    Comprueba si falló por timeout, respuesta `4xx`, respuesta `5xx` o error de conectividad.
  </Step>
  <Step title="Corrige el receptor">
    Ajusta URL, firma, autenticación interna o lógica de procesamiento en tu sistema.
  </Step>
  <Step title="Solicita reintento">
    Usa **Reintentar**. Bold vuelve a poner la notificación en `Pending` para envío inmediato.
  </Step>
</Steps>

<Warning>
  No reintentes varias veces sin corregir la causa. Si tu endpoint sigue fallando, generarás nuevos intentos fallidos.
</Warning>

## Detalles técnicos clave

| Tema | Detalle |
| --- | --- |
| Firma | Bold envía `X-HMAC-Signature` con HMAC-SHA256 del cuerpo JSON. |
| Timeout | Cada envío espera hasta 30 segundos. |
| Reintentos automáticos | Hasta 5 intentos con backoff exponencial. |
| Duplicados | Tu receptor debe deduplicar por `eventId`. |
| Trabajo pesado | Encola el evento y responde `2xx` rápido. |

## Problemas habituales

| Problema | Qué revisar |
| --- | --- |
| No aparece el evento esperado | Revisa que la suscripción use el evento y versión correctos. |
| La entrega queda en `Failure` | Abre los intentos y revisa código HTTP, cuerpo de respuesta o error. |
| Tu sistema procesa dos veces | Guarda `eventId` y haz el consumidor idempotente. |
| La firma no coincide | Verifica con el cuerpo crudo recibido, sin parsearlo ni reserializarlo. |
| No puedes crear suscripciones | Necesitas rol `App.Admin`. |
