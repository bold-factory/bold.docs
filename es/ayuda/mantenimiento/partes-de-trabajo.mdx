---
title: "Partes de trabajo"
description: "Registra correctivos y trabajos de mantenimiento con tiempos, acciones y materiales."
---

Un **parte de trabajo** registra una intervención sobre un activo. Explica qué se hizo, por qué se hizo, quién participó, cuánto tiempo se dedicó y qué materiales se consumieron.

## Tipos de parte

| Tipo | Uso |
| --- | --- |
| Correctivo | Responde a una incidencia, avería o necesidad no planificada. |
| Preventivo | Se genera o se crea para una revisión planificada. |

<Info>
  Necesitas `Maintenance.WorkOrders.Write` para crear, editar, completar, cancelar o registrar acciones, tiempos y materiales. Los resultados de acciones con imagen o archivo también requieren `Files.Files.Upload`.
</Info>

## Ciclo de vida

| Estado | Qué significa |
| --- | --- |
| Pendiente | El parte existe, pero no tiene tiempo activo. |
| En progreso | Hay un registro de tiempo activo. |
| En pausa | Los tiempos registrados están finalizados, pero el parte no está completado. |
| Completado | El trabajo queda cerrado y se bloquean cambios estructurales. |
| Cancelado | El parte queda anulado y se bloquean cambios estructurales. |

<Info>
  Al añadir un tiempo activo, el parte empieza o se reanuda. Cuando todos los tiempos terminan, queda en pausa. Si eliminas todos los tiempos, vuelve a pendiente.
</Info>

## Crear y ejecutar un parte

<Steps>
  <Step title="Crea el parte">
    Entra en **Mantenimiento** > **Partes de trabajo** y crea un parte con activo, tipo, causa principal y notas si aplica.
  </Step>
  <Step title="Añade causas auxiliares">
    Usa causas auxiliares solo cuando aporten análisis adicional. No pueden repetir la causa principal ni repetirse entre sí.
  </Step>
  <Step title="Añade acciones">
    Usa acciones de la causa o añade acciones manuales. Cada acción puede exigir un tipo de resultado distinto.
  </Step>
  <Step title="Registra tiempos">
    Añade tiempos por empleado. El primer tiempo puede asignar el empleado principal si el parte no lo tenía.
  </Step>
  <Step title="Consume materiales">
    Registra SKU, cantidad y lote cuando necesites trazabilidad y coste real.
  </Step>
  <Step title="Completa el parte">
    Revisa acciones, tiempos y materiales antes de completar. Al completar, Bold finaliza tiempos abiertos y bloquea cambios de cabecera, acciones, tiempos y materiales.
  </Step>
</Steps>

## Reglas importantes

| Área | Regla |
| --- | --- |
| Acciones | El resultado debe cumplir el tipo configurado: simple, sí/no, número, texto, imagen o archivo. |
| Acciones completadas | No se pueden completar dos veces. |
| Materiales | Las cantidades deben ser positivas. |
| Lotes | No puedes repetir el mismo lote en el detalle de un material. |
| Cantidad detallada | La suma por lotes no puede superar la cantidad total del material. |
| Parte cerrado | Completado o cancelado bloquea cambios estructurales. |
| Parte preventivo generado | Debe conservar programación y fecha de ocurrencia juntas. |

## Partes generados por preventivo

Un parte generado por una programación preventiva guarda dos referencias internas: la programación y la fecha de ocurrencia. Bold usa esas referencias para evitar duplicados y para decidir si una nueva ocurrencia debe omitirse.

Si cancelas un parte preventivo generado, la misma ocurrencia puede volver a generarse si el planificador la procesa otra vez. Si lo completas, queda como historial cerrado de esa ocurrencia.

## Eficiencia

Bold calcula eficiencia cuando existe tiempo teórico y tiempo real positivo:

```text
Eficiencia = tiempo teórico / tiempo real
```

Si una acción tenía `1 h` teórica y el técnico registra `1,25 h`, la eficiencia queda por debajo del 100 %. Usa esta métrica para comparar trabajos repetidos, no para evaluar casos excepcionales sin contexto.

## Problemas habituales

| Problema | Qué revisar |
| --- | --- |
| No puedes completar una acción | Falta el resultado esperado o el formato no coincide. |
| No puedes añadir materiales | El parte puede estar completado o cancelado. |
| El estado no cambia a pausa | Revisa si queda algún tiempo abierto. |
| El coste del parte no cuadra | Revisa lotes consumidos, cantidad, coste/hora de empleados y tiempos. |
| La eficiencia no aparece | Falta tiempo teórico o tiempo real positivo. |
