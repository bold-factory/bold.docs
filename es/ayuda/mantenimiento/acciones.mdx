title: "Acciones"

---

Las **acciones** son tareas reutilizables que puedes añadir a causas o partes de trabajo. Funcionan como checklist y permiten comparar intervenciones entre activos, turnos o técnicos.

## Qué define una acción

| Campo | Uso |
| --- | --- |
| Nombre | Texto que verá el operario en el parte. |
| Tiempo teórico | Duración esperada. Debe ser cero o positiva. |
| Tipo de resultado | Dato que debe registrar el operario al completar la acción. |
| Referencia externa | Código opcional para conectar con sistemas externos. |

## Tipos de resultado

| Tipo | Qué exige al completar |
| --- | --- |
| Simple | No exige resultado. |
| Sí/no | Exige un valor booleano. |
| Número | Exige un valor decimal. |
| Texto | Exige texto no vacío. |
| Imagen | Exige una URI segura `https`. |
| Archivo | Exige una URI segura `https`. |

<Info>
  Bold bloquea completar una acción ya completada. Si necesitas repetir el trabajo, añade otra acción al parte o crea un nuevo parte.
</Info>

## Crear una acción reutilizable

<Steps>
  <Step title="Abre Acciones">
    Entra en **Mantenimiento** y abre **Acciones**.
  </Step>
  <Step title="Nombra la tarea">
    Usa verbos claros, por ejemplo `Cambiar aceite`, `Comprobar presión` o `Limpiar filtro`.
  </Step>
  <Step title="Define tiempo teórico">
    Registra la duración esperada para poder calcular eficiencia cuando el parte tenga tiempo real.
  </Step>
  <Step title="Elige tipo de resultado">
    Usa el tipo mínimo que capture el dato necesario. No pidas archivo o imagen si basta con un sí/no.
  </Step>
  <Step title="Guarda y reutiliza">
    Añade la acción a causas frecuentes o directamente a partes de trabajo.
  </Step>
</Steps>

## Ejemplos

| Acción | Tipo recomendado | Motivo |
| --- | --- | --- |
| Cambiar correa | Simple | Basta con confirmar que se completó. |
| Comprobar presión | Número | Necesitas conservar el valor medido. |
| Verificar fuga | Sí/no | El resultado se interpreta como conforme o no conforme. |
| Adjuntar foto de daño | Imagen | El resultado necesita evidencia visual. |

## Problemas habituales

| Problema | Qué revisar |
| --- | --- |
| El operario no puede completar la acción | Revisa el tipo de resultado y el formato introducido. |
| La eficiencia no se calcula | Falta tiempo teórico o tiempo real en el parte. |
| El checklist queda demasiado largo | Mueve acciones poco frecuentes a causas específicas. |
| No puedes eliminar una acción | Revisa si ya está en uso o si fue eliminada antes. |
