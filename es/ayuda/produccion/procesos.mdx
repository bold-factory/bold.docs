Los **procesos** son categorías reutilizables de trabajo productivo, como corte, montaje, soldadura, pintura, inspección o embalaje.

Úsalos para conectar una operación de receta con las estaciones capaces de ejecutarla. Cuando una operación necesita un proceso, Bold puede ayudarte a elegir recursos compatibles y calcular planificación con una eficiencia realista.

## Antes de empezar

Define procesos al nivel en que planificas y analizas planta. No crees un proceso para cada microtarea si no cambia la estación, la duración, la capacidad o el análisis.

## Crear un proceso

<Steps>
  <Step title="Abre procesos">
    Entra en **Fabricación > Procesos**.
  </Step>
  <Step title="Crea el proceso">
    Indica un nombre que el equipo reconozca, como **Corte**, **Montaje** o **Control final**.
  </Step>
  <Step title="Asigna estaciones compatibles">
    Relaciona las estaciones que pueden ejecutar ese proceso.
  </Step>
  <Step title="Define eficiencia">
    Ajusta la eficiencia por estación si una máquina es más lenta, más rápida o menos adecuada.
  </Step>
  <Step title="Revisa herramientas y características">
    Añade requisitos cuando el proceso necesite herramientas concretas o capacidades especiales.
  </Step>
</Steps>

## Estaciones y eficiencia

Cada proceso puede tener varias estaciones compatibles. Solo una debe actuar como estación principal. Si marcas una nueva estación como principal, la anterior pasa a ser auxiliar.

La eficiencia debe ser positiva. Afecta a duración prevista, coste estimado y secuenciación.

Los tiempos estándar se definen para una eficiencia del 100%. Si una operación dura 60 minutos al 100% y la estación compatible tiene eficiencia del 75%, Bold debe planificar más tiempo para esa estación.

| Caso | Configuración recomendada |
| --- | --- |
| Dos estaciones hacen el mismo trabajo | Asigna ambas al proceso. |
| Una estación es más lenta | Usa una eficiencia menor para esa estación. |
| Una estación debe usarse por defecto | Márcala como principal. |
| El trabajo requiere una herramienta concreta | Modela una característica y herramientas compatibles. |

## Herramientas y características

Los procesos pueden relacionarse con características para expresar requisitos de ejecución. Las herramientas también tienen características, y esa relación ayuda a seleccionar una herramienta compatible cuando una operación lo exige.

| Elemento | Uso |
| --- | --- |
| Característica | Requisito técnico, formato, útil o capacidad necesaria. |
| Herramienta | Recurso físico que cumple una o varias características. |
| Proceso | Tipo de trabajo que puede necesitar estaciones y herramientas compatibles. |

## Problemas habituales

- Si una estación no aparece al planificar, revisa que esté asignada al proceso.
- Si la duración prevista no encaja, revisa eficiencia y tiempo teórico.
- Si una operación exige herramienta, revisa características y herramientas compatibles.
- Si el proceso es demasiado específico, puede fragmentar la planificación y los KPIs.
