---
title: "Condicionar operaciones, entradas, pasos y controles"
description: "Activa operaciones, materiales, pasos o controles solo cuando se cumplan condiciones definidas."
---

Las **condiciones** te permiten diseñar una única ruta de fabricación para múltiples variantes de un mismo producto. En lugar de crear rutas distintas para cada modelo, el sistema muestra u oculta elementos en la pantalla del operario basándose en las propiedades del artículo que se está fabricando.

Existen **cuatro casos de uso clave** para aplicar estas condiciones en tu planta:

## Condicionar operaciones

Cuando una fase entera del proceso de producción aplica únicamente a ciertas variantes del artículo.

<Note>
  **Ejemplo:** la operación de _Pintura_ solo debe activarse si la propiedad _Acabado_ del producto requiere pintura. Si el cliente pide un cuadro de bicicleta con acabado en aluminio pulido, el sistema omite esta etapa.
</Note>

## Condicionar entradas

Cuando un material o insumo específico solo se consume en algunas variantes del producto.

<Note>
  **Ejemplo:** configuras dos entradas de material en la misma operación: una para _Pintura roja_ y otra para _Pintura azul_. Al condicionar cada entrada a la propiedad _Color_, la OF solo cargará, mostrará y descontará del inventario el color que requiere esa bicicleta.
</Note>

## Condicionar pasos

Cuando una instrucción de trabajo dentro de una operación solo debe realizarse para ciertas variantes.

<Note>
  **Ejemplo:** en la etapa de ensamblaje, decides que solo las bicicletas con la propiedad **acabado premium** requieran un paso adicional de _Aplicación de cera protectora_. Para los modelos estándar, este paso no aparecerá en la pantalla del operario.
</Note>

## Condicionar controles

Sirve para que el sistema adapte la inspección al producto exacto que está en la máquina.

<Note>
  **Ejemplo:** una prueba de presión de bomba de agua a 15 bares solo se activa si el **Modelo** es igual a `Profesional / Hostelería`. Así, el operario solo realiza esta prueba cuando pasa una máquina destinada a cafeterías.
</Note>

## Añadir una condición

**Origen (Propiedad / Contexto de la OF):** le indica al sistema dónde tiene que ir a buscar la información para validar la regla.

- **Propiedad:** se refiere a las características técnicas fijas del artículo (ej. el material del cuadro: Aluminio, el color: Rojo, el tamaño: XL).
- **Contexto de la OF (Orden de Fabricación):** se refiere a los datos dinámicos de ese producto en particular (ej. el nombre del cliente, la cantidad total a fabricar, si es un pedido urgente).

**Operador:** la lógica matemática que conecta la propiedad con el resultado esperado (ej. igual a, distinto de, contiene, empieza por o termina por). 

**Valor/Opción:** el sistema te pedirá el valor final (ej. Propiedad: material / Operador: igual a / Valor: carbono).
