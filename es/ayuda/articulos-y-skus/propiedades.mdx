title: "Propiedades"
description: "Define atributos reutilizables para clasificar productos, validar artículos y usar variantes en recetas."

Las propiedades son atributos reutilizables para describir artículos y variantes de producto. Ejemplos habituales son color, material, largo, acabado, tipo de envase o fecha de caducidad técnica.

Primero creas la propiedad global. Después la asignas a los productos que la necesitan. Los artículos de esos productos deben informar valores coherentes para quedar bien clasificados y validados.

## Cuándo usar propiedades

Usa propiedades cuando una característica debe servir para filtrar, validar, fabricar, planificar o reutilizar valores entre muchos artículos.

No uses propiedades para notas ocasionales. Si el valor no se va a filtrar ni usar en lógica de producto, un campo descriptivo suele ser suficiente.

## Clases de propiedad

| Clase | Úsala para | Ejemplo |
| --- | --- | --- |
| Selección de opción | Listas controladas y cerradas. | Color, acabado, familia logística. |
| Sí/no | Características binarias. | Es inflamable, requiere frío. |
| Número entero | Cantidades sin decimales. | Número de capas, calibre entero. |
| Número con decimales | Medidas o parámetros con precisión. | Espesor, densidad, longitud. |
| Fecha/hora | Fechas relevantes para clasificación o control. | Fecha de certificación, revisión técnica. |
| Texto libre | Valores técnicos que no encajan en una lista cerrada. | Código interno, observación corta. |

<Warning>
  Elige bien la clase antes de usar la propiedad en muchos productos. La clase define el tipo de valor que podrán guardar los artículos.
</Warning>

## Selección de opción

Usa **Selección de opción** cuando quieres limitar los valores posibles a una lista controlada. Las opciones pertenecen a la propiedad global, no a un producto concreto.

Ejemplo:

- Propiedad: **Color**.
- Opciones: **Rojo**, **Azul**, **Negro**.
- Productos que la usan: envases, etiquetas y piezas pintadas.

Si cada producto necesita una lista distinta, crea propiedades distintas o usa una clase más flexible. No mezcles listas incompatibles en una sola propiedad global.

## Crear una propiedad

<Steps>
  <Step title="Crea la propiedad global">
    Ve a **Catálogo > Propiedades** y crea la propiedad con **Nombre**, **Clase** y **Referencia externa** si viene de otro sistema.
  </Step>
  <Step title="Añade opciones si corresponde">
    Si la clase es **Selección de opción**, añade las opciones permitidas.
  </Step>
  <Step title="Asigna la propiedad al producto">
    En la ficha del producto, añade la propiedad para que sus artículos puedan usarla.
  </Step>
  <Step title="Rellena valores en los artículos">
    Informa el valor de cada artículo. Revisa los artículos existentes después de asignar una propiedad nueva.
  </Step>
  <Step title="Usa la propiedad en procesos">
    Filtra artículos, prepara variantes y usa propiedades en condiciones de receta cuando aplique.
  </Step>
</Steps>

## Qué cambia en Bold

Al asignar o quitar una propiedad de un producto, Bold revisa el estado de validación de los artículos de ese producto. Esto ayuda a detectar artículos que todavía no tienen valores completos o que ya no cumplen la estructura del producto.

Las propiedades también aparecen como datos de referencia para:

- Buscar y filtrar artículos.
- Diferenciar variantes dentro de un producto.
- Configurar condiciones de receta basadas en atributos.
- Definir valores fijos de semielaborados cuando una receta depende de propiedades.
- Mantener consistencia en importaciones e integraciones.

## Copiar propiedades

Puedes copiar una propiedad de tipo **Selección de opción** para crear una nueva lista con las mismas opciones iniciales. Esto sirve cuando necesitas partir de una lista existente, pero mantenerla independiente a partir de ese momento.

Ejemplo: copias **Color comercial** para crear **Color técnico**. Después puedes ajustar opciones sin mezclar usos comerciales y de fabricación.

## Importaciones e integraciones

Usa **Referencia externa** en propiedades y opciones cuando los valores llegan de otro sistema. Mantén esas referencias estables para que las importaciones actualicen el valor correcto.

Buenas prácticas:

- Usa opciones controladas para valores que deben coincidir exactamente.
- Evita texto libre si luego necesitarás agrupar o fabricar por ese valor.
- No reutilices la misma propiedad para significados distintos.
- Revisa artículos incompletos después de asignar una propiedad nueva a un producto con variantes existentes.

## Ejemplos

| Necesidad | Configuración recomendada |
| --- | --- |
| Filtrar pinturas por acabado | Propiedad **Acabado** como **Selección de opción**. |
| Guardar longitud de perfiles | Propiedad **Longitud** como **Número con decimales**. |
| Indicar si un material requiere lote | Propiedad **Requiere lote** como **Sí/no** si solo clasifica; usa la configuración de lotes del artículo para controlar stock. |
| Elegir insumos según color del producto final | Propiedad **Color** asignada al producto y usada en condiciones de receta. |
| Importar valores desde ERP | Referencia externa en propiedad y opciones. |

## Permisos

| Acción | Permiso necesario |
| --- | --- |
| Ver propiedades y opciones | `Items.Properties.Read` |
| Crear, editar, copiar o eliminar propiedades | `Items.Properties.Write` |
| Asignar propiedades a productos | `Items.Products.Write` |
| Editar valores de artículo | `Items.Skus.Write` |

## Problemas habituales

| Problema | Qué revisar |
| --- | --- |
| Una propiedad no aparece en el artículo | Comprueba que esté asignada al producto del artículo. |
| No puedes añadir opciones | Solo las propiedades de clase **Selección de opción** aceptan opciones. |
| Los artículos quedan incompletos | Rellena valores para las propiedades requeridas por el producto. |
| Una importación no encuentra el valor | Revisa la referencia externa de la propiedad y de la opción. |
| La lista de opciones se volvió confusa | Divide la propiedad si estás mezclando significados distintos. |
