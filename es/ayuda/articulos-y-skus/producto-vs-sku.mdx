La diferencia entre **Producto** y **Artículo** es una de las decisiones más importantes del catálogo.

Un **Producto** agrupa variantes. Un **Artículo** representa la unidad física concreta que compras, fabricas, almacenas, consumes o vendes.

## Comparación rápida

| Concepto | Qué representa | Ejemplo |
| --- | --- | --- |
| Producto | Familia o concepto comercial | Mesa Nórdica |
| Propiedad | Atributo que diferencia variantes | Color, largo, acabado |
| Artículo | Variante física concreta | Mesa Nórdica blanca 120 cm |
| Lote | Conjunto trazable de unidades de un artículo | Lote fabricado el 2026-06-15 |

## Cuándo usar un producto con varios artículos

Usa un producto con varios artículos cuando las variantes:

- tienen stock separado;
- se compran, fabrican o venden por separado;
- tienen propiedades importantes para el cliente o la operación;
- pueden tener costes, lotes o recetas distintas;
- necesitan trazabilidad independiente.

## Cuándo un producto puede tener un solo artículo

Un producto puede tener un único artículo si no existen variantes físicas o si la empresa no necesita separarlas. Por ejemplo, un repuesto estándar comprado siempre igual.

<Tip>
  Empieza con la estructura más simple que mantenga stock y trazabilidad correctos. Añade propiedades cuando la diferencia tenga impacto operativo.
</Tip>

## Errores habituales

### Crear productos para cada variante

Si creas un producto por variante, pierdes agrupación. Será más difícil consultar actividad conjunta, reutilizar recetas o definir propiedades comunes.

### Crear un solo artículo para variantes distintas

Si usas un único artículo para variantes diferentes, mezclarás stock. Esto puede ocultar roturas reales, errores de picking o diferencias de coste.

### Usar propiedades como texto libre

Las propiedades deben explicar variantes controladas. Si necesitas una nota puntual, usa descripción o notas operativas.

<Warning>
  No debería haber dos artículos activos con el mismo producto y los mismos valores de propiedades. Si ocurre, revisa si son duplicados o si falta una propiedad.
</Warning>
