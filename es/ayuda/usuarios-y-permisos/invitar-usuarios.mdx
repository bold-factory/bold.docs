Un usuario es la cuenta con la que una persona entra en Bold. Invita usuarios cuando necesiten acceso propio a la aplicación, o a funciones administrativas.

<Info>
  Usuario y empleado no son lo mismo. El usuario controla acceso y permisos. El empleado representa a una persona operativa para turnos, trabajo, costes o asignaciones.
</Info>

## Antes de invitar

- Ten claro qué módulos debe usar la persona.
- Crea o revisa el rol que le vas a asignar.
- Decide si debe vincularse a un empleado operativo.
- Evita compartir una cuenta entre varias personas si necesitas auditoría.

## Datos del usuario

| Campo | Para qué sirve |
| --- | --- |
| Email | Identifica la cuenta y se usa para iniciar sesión. |
| Nombre y apellidos | Ayudan a reconocer a la persona en listas y auditoría. |
| Empleado | Vincula la cuenta con una persona operativa para trabajo, turnos y asignaciones. |
| Roles | Definen qué módulos ve y qué acciones puede ejecutar. |

## Invitar a una persona

<Steps>
  <Step title="Abre **Panel de control > Usuarios**">
    Entra con un usuario con rol `App.Admin`.
  </Step>
  <Step title="Crea el usuario">
    Introduce email, nombre, apellidos y empleado si aplica.
  </Step>
  <Step title="Asigna roles">
    Selecciona uno o varios roles que reflejen sus responsabilidades reales.
  </Step>
  <Step title="Vincula empleado si aplica">
    Relaciona el usuario con el empleado operativo cuando deba registrar trabajo o actuar en planta.
  </Step>
  <Step title="Confirma el acceso">
    Pide a la persona que inicie sesión y comprueba que ve solo los módulos necesarios.
  </Step>
</Steps>

## Vincular usuario y empleado

Vincula el usuario con un empleado cuando sus acciones deban asociarse a una persona operativa. Esto importa en planta, turnos, costes, asignaciones y tareas.

Casos habituales:

- Un operario que registra trabajo desde una tablet.
- Un responsable que actúa en nombre de un empleado en un flujo operativo.
- Una cuenta que necesita consultar datos, pero no registrar trabajo. En este caso puede no necesitar vínculo con empleado.

Revisa la vinculación cuando una persona cambia de puesto o cuando se reutiliza un dispositivo compartido.

## Recomendaciones

| Situación | Recomendación |
| --- | --- |
| Nuevo responsable de almacén | Asigna permisos de lectura y escritura de almacén. |
| Operario de planta | Asigna permisos de trabajo de producción y vínculo con empleado. |
| Planificador | Asigna lectura y escritura de planificación, compras y producción según proceso. |
| Integración técnica | Usa una credencial técnica gestionada por un administrador en vez de una cuenta personal. |
| Usuario temporal | Usa roles mínimos y define una fecha de revisión del acceso. |

<Warning>
  Retira o ajusta roles cuando una persona cambia de responsabilidad. Mantener accesos antiguos aumenta el riesgo operativo.
</Warning>

## Problemas habituales

| Problema | Qué revisar |
| --- | --- |
| No puedes crear el usuario | Necesitas rol `App.Admin`. |
| La persona entra, pero no ve módulos | Falta acceso común o permisos de lectura. |
| Puede ver, pero no guardar | Falta permiso de escritura o acción específica. |
| Las acciones no aparecen asociadas al empleado correcto | Revisa el vínculo usuario-empleado. |
| Conserva accesos antiguos | Quita roles que ya no correspondan a su responsabilidad actual. |
